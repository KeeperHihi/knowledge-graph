# 答辩演示顺序

这个文档是我给自己准备的现场演示稿，不是完整说明书。真正答辩时，我不想一开始就只展示一张图，因为那样老师可能会觉得图谱是直接写出来的。所以我的顺序是：先证明数据来源，再证明中间过程，最后用网页五个页签把整条链路讲顺。

## 1. 先讲数据从哪来

先打开 `data/raw/` 和 `data/raw/source_manifest.json`。

我会先说：

- 我没有直接手写结构化知识图谱，而是先整理了 16 份图灵相关的 raw 文本。
- 每份 raw 文本都在 `source_manifest.json` 里记录了来源标题和链接，后面的事件、关系和证据句都能回到这些文本。
- 文本是根据公开资料整理成适合课程作业展示的短句，不是照搬网页全文。

这里不用讲太久，重点是把“从 raw_data 抽取，而不是直接编图谱”这件事说清楚。

## 2. 再讲中间结果

第二步打开这几个文件：

- `data/intermediate/mentions.jsonl`
- `data/intermediate/linked_entities.jsonl`
- `data/output/report.json`
- `data/output/evaluation_summary.json`

我会强调：这个项目不是从原文直接跳到最终图谱，中间确实有实体抽取、实体消歧、事件抽取和关系抽取。

这里可以拿“剑桥”举例：

- 在 `mentions.jsonl` 里，它先只是一个从句子里抽到的 mention。
- 到 `linked_entities.jsonl` 里，它会有候选实体和打分。
- 如果上下文是“学习数学、大学”，就更像 `University of Cambridge`；如果上下文是“城市、英格兰东部”，就更像 `Cambridge` 这个地点。

我还会顺手说一句：`evaluation_summary.json` 是我做的小样本人工核对，目前实体消歧、关系抽取、事件抽取都是 `8/8`。这个样本不大，但能说明我不是只看页面能不能跑。

## 3. 先看输出文件，再打开网页

网页演示前，我会快速打开：

- `data/output/events.json`
- `data/output/relations.csv`
- `data/output/traceability.json`
- `data/output/explainability.json`

这里我准备讲的重点是：我没有直接从句子硬抽三元组，而是先抽事件，再从事件生成关系。比如论文发表句会先形成 `PublicationEvent`，再生成 `Alan Turing - published - Computable Numbers`。

`traceability.json` 用来回答“这条关系从哪来”，`explainability.json` 用来支撑网页里的四个过程案例。这样老师问某条边的来源时，我能回到具体文本和证据句。

## 4. 按五个页签讲网页

正式打开网页前，我会先跑一次网页材料自检：

```bash
python3 scripts/check_web_demo.py --port 8765
```

这个命令不是新的功能演示，只是帮我确认首页、五个页签、图谱数据和 explainability 案例都能正常访问，避免到答辩现场才发现数据没生成好。

启动网页：

```bash
python3 scripts/run_visualization.py --port 8000
```

然后打开：

```text
http://127.0.0.1:8000/web/index.html
```

网页我会按从左到右的五个页签讲。

### 实体抽取

先停在 `实体抽取` 页签。

我会指着高亮句子说：这里展示的是从 raw 句子里找 mention 的过程。比如 `Alan Turing` 是词典匹配，`1936` 是正则匹配，`《Computable Numbers》` 是书名格式规则。这个页面的作用不是炫界面，而是让老师直接看到“原文片段 -> mention”的第一步。

### 实体消歧

再点 `实体消歧` 页签。

这里重点讲候选打分表。我的消歧方法很简单，就是：

```text
score = 0.5 * alias + 0.3 * context + 0.2 * type
```

我会用 `Cambridge` 或 `剑桥大学` 解释：别名像不像是一部分，上下文里有没有“学习、大学、城市”这些关键词也是一部分，抽取阶段给出的类型也会影响分数。最后不是我手动指定结果，而是候选实体按这个分数排序。

### 事件抽取

接着点 `事件抽取` 页签。

我会按页面上的三步讲：

1. 先找触发词，比如“发表”“学习”“工作”“参与”。
2. 再找参与实体，比如人物、机构、作品、设备。
3. 最后形成一个事件记录，比如 `PublicationEvent` 或 `EducationEvent`。

这里要强调事件层的价值：如果直接生成三元组，解释会比较跳；先有事件，关系就有了中间依据。

### 关系抽取

再点 `关系抽取` 页签。

这一页讲“事件或句式规则怎么变成三元组”。我会照着页面说：先确定头实体，再根据事件类型或触发词确定关系，再找尾实体，最后得到一条三元组，并保留证据句。

例如：

```text
Alan Turing - published - Computable Numbers
```

这条关系不是凭空写的，它来自发表论文那句 raw 文本，也能回到对应事件。

### 知识图谱

最后点 `知识图谱` 页签。

这里我会演示四个交互：

1. 点 `Alan Turing`，看右侧显示节点信息和它的直接连边。
2. 点 `Joan Clarke` 或 `Max Newman`，说明这张图不是只有图灵一个主体，也有相关人物自己的关系。
3. 点 `Bombe` 或 `Manchester Mark I`，说明设备节点不是孤立的，右侧能看到它们的相邻关系。
4. 拖动一个节点，说明页面可以手动整理局部图，方便现场看清关系。

如果图里标签有一点重叠，我不会把它说成 bug，而是说明力导向图初始布局会有局部重叠，所以我加了拖动节点的交互来手动调整。

## 5. 只剩 1 分钟时怎么讲

如果现场时间很短，我就不打开太多文件，只走这条路线：

1. 打开 `data/raw/source_manifest.json`，说明数据有来源。
2. 打开网页，按五个页签各讲一句：
   - 实体抽取：从 raw 句子高亮 mention。
   - 实体消歧：用候选打分解决 Cambridge 这类歧义。
   - 事件抽取：用触发词和参与实体形成事件。
   - 关系抽取：从事件生成三元组并保留证据句。
   - 知识图谱：点击节点看连边，拖动节点整理布局。
3. 最后打开 `evaluation_summary.json`，说人工核对样本目前三项都是 `8/8`。

这个版本讲得很快，但至少能覆盖老师最关心的“数据来源、四个功能、可视化、可解释性”。

## 6. 最后收尾

最后我会主动承认局限：

- 这是规则法，不是通用模型，所以换到别的领域需要重新补词典和触发词。
- 代词、省略和复杂长句仍然可能漏抽。
- 小样本人工评测通过，不代表所有图灵资料都能完美覆盖。

但我也会强调它适合作为本科课程作业：没有调用大模型或预训练模型，过程清楚，数据从 raw 文本来，每一步都有中间结果，网页也能把四个功能和最终图谱连起来讲。
